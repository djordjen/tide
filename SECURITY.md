# Security Policy

This file covers vulnerability reporting for the TIDE codebase. The framework's
application authorization design is documented separately in
[`docs/SECURITY.md`](docs/SECURITY.md).

## Reporting a vulnerability

Do not open a public issue for a suspected vulnerability. Use the repository's
private security-advisory facility or another private channel published by the
maintainer. Include affected versions, reproduction steps, expected impact, and
any known mitigations. Avoid accessing data that you do not own while testing.

Until a first supported release is published, there is no supported-version
matrix or response-time guarantee. Security fixes should include a regression
test and should avoid disclosing protected values in diagnostics or logs.
